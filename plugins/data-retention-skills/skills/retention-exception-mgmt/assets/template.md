# Retention Exception Management Templates — Orion Data Vault Corp

## Retention Exception Request Template

```
RETENTION EXCEPTION REQUEST
Reference: RET-EXC-YYYY-NNNN
Date: YYYY-MM-DD
Requestor: [Name, Title, Department]

SECTION 1: DATA IDENTIFICATION
- Data Category: [CAT-XXX]
- Specific data set: [Description]
- Records affected: [Count]
- Current retention period: [Per schedule]
- Scheduled deletion date: [YYYY-MM-DD]

SECTION 2: EXCEPTION REQUEST
- Extension period: [X days/months]
- Proposed new deletion date: [YYYY-MM-DD]
- Exception category: [Select from approved categories]

SECTION 3: JUSTIFICATION
[Detailed explanation — minimum 200 words]

SECTION 4: PROPORTIONALITY
- Full data set needed? [Yes/No — explain]
- Pseudonymization possible? [Yes/No — explain]
- Access restrictions applied? [Detail]
- Data sensitivity: [Level]

SECTION 5: SUPPORTING DOCUMENTATION
[Attach evidence]

Requestor Signature: _________________ Date: _________
```

## DPO Assessment Template

```
DPO ASSESSMENT — RETENTION EXCEPTION
Reference: RET-EXC-YYYY-NNNN

NECESSITY TEST:
□ Exception genuinely necessary
□ Specific consequences of deletion demonstrated
□ Exception category legitimate

PROPORTIONALITY TEST:
□ Duration proportionate
□ Data minimization applied
□ Access restrictions adequate
□ Pseudonymization applied where feasible

LEGAL BASIS CHECK:
□ Valid basis for continued processing
□ Data subjects informed (if consent-based)
□ No conflicting data subject rights

RISK ASSESSMENT:
- Risk to data subjects: [Low/Medium/High]
- Risk from not retaining: [Low/Medium/High]

RECOMMENDATION:
□ APPROVE — as requested
□ APPROVE WITH CONDITIONS
□ APPROVE REDUCED DURATION
□ REJECT

DPO: _________________ Date: _________
```
