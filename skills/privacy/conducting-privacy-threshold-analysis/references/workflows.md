# PTA Workflows

## Workflow 1: PTA Execution
1. Processing owner completes PTA questionnaire (Sections A, B, C).
2. DPO reviews responses and validates the determination.
3. DPO signs off on the PTA.
4. PTA is logged in the PTA register.
5. If DPIA required: initiate DPIA process.
6. If DPIA not required: approve processing with documentation.

## Workflow 2: PTA Integration with Change Management
1. All new projects, system procurements, and processing changes must trigger a PTA.
2. IT change management system includes PTA as a mandatory gate.
3. Procurement processes require PTA for any vendor processing personal data.
4. PTA re-screening required when processing scope changes after initial approval.
