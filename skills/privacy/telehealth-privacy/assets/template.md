### Platform Security Requirements
- [ ] End-to-end encryption (AES-128+ / AES-256 preferred)
- [ ] TLS 1.2+ for all connections
- [ ] No unencrypted fallback
- [ ] Unique user authentication for providers
- [ ] Patient identity verification
- [ ] MFA for provider access
- [ ] Automatic session termination
- [ ] Session event logging
- [ ] Recording access audit logging
- [ ] Tamper-resistant logs
- [ ] 6-year log retention

## Provider Environment

- [ ] Private room with closed door required
- [ ] No visible patient information behind provider
- [ ] Headphones/earbuds required
- [ ] Only managed/approved devices permitted
- [ ] No public locations for telehealth visits
- [ ] Workstation lock when stepping away

## Remote Patient Monitoring

- [ ] RPM vendor BAAs in place
- [ ] Patient informed consent obtained
- [ ] Device-to-cloud encryption verified
- [ ] Data collection limited to clinically necessary
- [ ] Minimum necessary notifications (minimize PHI content)
