## How to Get Added as a Contributor

**Automatic recognition** -- Any merged pull request triggers contributor addition. You do not need to ask.

**Manual request** -- If your contribution was not code-based (e.g., you reported a critical regulatory inaccuracy via an issue, or helped with research in a discussion), open an issue with the title:

```
[Contributor] Add @your-username for [contribution type]
```

Example:

```
[Contributor] Add @janedoe for research, review
```

A maintainer will add you to the table above within one release cycle.

### What Counts as a Contribution

- Submitting a new privacy skill via pull request
- Fixing regulatory inaccuracies or updating enforcement precedents
- Improving Python scripts or adding validation tooling
- Translating skills or documentation
- Reporting bugs or regulatory errors via GitHub Issues
- Proposing new skill ideas or subdomain coverage
- Reviewing pull requests for technical or legal accuracy
- Writing or improving documentation
- Sharing the project with the privacy community

### Attribution Format

Contributors are listed with:
- GitHub avatar (linked to profile)
- Display name and GitHub handle
- Project role (Contributor, Reviewer, Translator, etc.)
- Contribution type emojis matching the legend above

## Recognition in Releases

Contributors are mentioned in [CHANGELOG.md](CHANGELOG.md) release notes for the specific version(s) they contributed to. Major contributors may be invited to join the maintainer team.

## All Contributors Specification

This project follows the [All Contributors](https://allcontributors.org/) specification v7.0. The specification provides a standard for recognizing all contributions, including those that do not involve pushing code. For full details, visit [allcontributors.org/docs/en/specification](https://allcontributors.org/docs/en/specification).
